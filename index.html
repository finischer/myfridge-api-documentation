API Referenz

Die myFridge-API ist um REST herum organisiert. Unsere API hat vorhersehbare ressourcenorientierte URLs, gibt JSON-codierte Antworten zurück und verwendet standardmäßige HTTP-Antwortcodes.

Die API empfängt Daten im JSON-Format und verarbeitet die Anfrage anschließend. Je nach Anfrage, müssen bestimmte Parameter mitgegeben werden.

BASE URL
https://myfridge-backend.herokuapp.com/




Errors

myFridge verwendet herkömmliche HTTP-Antwortcodes, um den Erfolg oder Misserfolg einer API-Anfrage anzuzeigen. Im Allgemeinen: Codes im 2xx-Bereich zeigen einen Erfolg an. Codes im 4xx-Bereich zeigen einen Fehler an, der angesichts der bereitgestellten Informationen fehlgeschlagen ist (z. B. ein erforderlicher Parameter wurde ausgelassen, eine Aufladung ist fehlgeschlagen usw.). Codes im 5xx-Bereich weisen auf einen Fehler mit den Servern von myFridge hin (diese sind selten).

Einige 4xx-Fehler, die programmatisch behandelt werden können, enthalten einen Fehlercode, der den gemeldeten Fehler kurz erklärt.


HTTP STATUS CODE SUMMARY
200 - OK Everything worked as expected.
400 - Bad Request The request was unacceptable, often due to missing a required parameter.
401 - Unauthorized No valid API key provided.
402 - Request Failed The parameters were valid but the request failed.
403 - Forbidden The API key doesn't have permissions to perform the request.
404 - Not Found The requested resource doesn't exist.
409 - ConflictThe request conflicts with another request (perhaps due to using the same idempotent key).
429 - Too Many RequestsToo many requests hit the API too quickly. We recommend an exponential backoff of your requests.
500, 502, 503, 504 - Server Errors Something went wrong on myFridge's end. (These are rare.)




User

Retrieve a User

METHOD: GET

Um einen User zu bekommen, können Sie nach einem bestimmten User-Object suchen.


Parameter

_id required in url

Die ObjectID des Users.





Update a User

METHOD: PUT

Um bestimmte Daten eines Users ändern zu können, können Sie bestimmte Attribute des jeweiligen User-Objects updaten. Auch wenn nicht alle Daten zeitgleich geändert werden, müssen alle Attribute mitgegeben werden.


Parameter

_id required in url

Der User, bei der die Daten geändert werden sollen.

vorname required

Der neue Vorname des Users.

nachname required

Der neue Nachname des Users.

passwort required

Das neue Passwort.

email required

Die neue E-Mail Adresse des Users.





Delete a User

METHOD: DELETE

Um einen bestimmten User zu löschen, können Sie nach einem bestimmten User-Object suchen und ihn dadurch löschen. Alle Attribute (inklusive der $oid) werden gelöscht.


Parameter

_id required in url

Der User, welcher gelöscht wird.





List all Users

METHOD: GET

Gibt eine Liste der User zurück, die Sie zuvor erstellt haben. Die User werden in sortierter Reihenfolge zurückgegeben, wobei die ältesten User zuerst angezeigt werden.


Parameter

Keine Parameter notwendig.





Anmeldung

Die "Session"-Logik besteht darin, dass für einen User, ein Token generiert wird, der für 30 Minuten gültig ist.

Dieser wird bei jeder Abfrage mitgegeben, um Funktionen nutzen zu können, die nur beim Status "eingeloggt" möglich sind.

ENDPOINTS

POST
             /api/login
GET
               /api/logout

Update a Product in a Shoppinglist

METHOD: PUT

Um Daten eines Produktes zu ändern, können Sie die ID der Shoppingliste in der URL angeben und die neuen Daten als JSON-body an den Endpunkt senden. Wenn der Status eines Produktes von „offen“ zu „besorgt“ geändert wird, wird er automatisch dem Kühlschrank des Users hinzugefügt. Wenn der Status von „besorgt“ zu „offen“ geändert wird, wird das Produkt aus dem Kühlschrank des Users entfernt. Wenn die Menge 0 entspricht, wird das Produkt aus der Shoppingliste entfernt, ansonsten wird nur die Menge angepasst.


Parameter

_id required in url

Der eindeutige Bezeichner der Einkaufsliste

name required

Der Name des Produkts

menge required

Die Menge des Produkts

status required

Der Status des Produktes, entweder „offen“ oder „besorgt“.

Recipe

Recipe-Objects repräsentieren ein Rezept der myFridge-App. Ein solches Objekt enthält zum einen Variablen, wie den Rezeptnamen, die Zubereitungszeit und einen Schwierigkeitsgrad, zum anderen die Dictionaries „Zutaten“ und „Zubereitung“. Im Dictionary „Zutaten sind alle benötigten Produkte mit Mengenangaben und Einheit gespeichert, in „Zubereitung“ sind die einzelnen Zubereitungsschritte enthalten. Mit der API können Sie Rezepte erstellen, löschen und aktualisieren. Sie können sowohl einzelne Rezepte als auch eine Liste aller Rezepte abrufen.

ENDPOINTS

Search by Category

METHOD: GET

Wenn Sie Produkte nach Kategorie suchen wollen, können Sie das mit dem Endpoint /api/fridgesearch/{fridge_id}/{categorie} machen.


Parameter

fridge_id required in url

Der eindeutige Bezeichner des Kühlschranks.

categorie required in url

Die Kategorie, nach der gesucht werden soll.





Search in Fridge

METHOD: GET

Wenn Sie nach Produkte innerhalb eines Kühlschranks suchen wollen, können Sie das mit dem Endpoint /api/searchresult/{fridge_id}/{search_string} machen.


Parameter

fridge_id required in url

Der eindeutige Bezeichner des Kühlschranks.

search_string required in url

Der eingegeben Wert, nach dem gesucht werden soll.
