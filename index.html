API Referenz

Die myFridge-API ist um REST herum organisiert. Unsere API hat vorhersehbare ressourcenorientierte URLs, gibt JSON-codierte Antworten zurück und verwendet standardmäßige HTTP-Antwortcodes.

Die API empfängt Daten im JSON-Format und verarbeitet die Anfrage anschließend. Je nach Anfrage, müssen bestimmte Parameter

BASE URL
https://myfridge-backend.herokuapp.com/




Errors

myFridge verwendet herkömmliche HTTP-Antwortcodes, um den Erfolg oder Misserfolg einer API-Anfrage anzuzeigen. Im Allgemeinen: Codes im 2xx-Bereich zeigen einen Erfolg an. Codes im 4xx-Bereich zeigen einen Fehler an, der angesichts der bereitgestellten Informationen fehlgeschlagen ist (z. B. ein erforderlicher Parameter wurde ausgelassen, eine Aufladung ist fehlgeschlagen usw.). Codes im 5xx-Bereich weisen auf einen Fehler mit den Servern von myFridge hin (diese sind selten).

Einige 4xx-Fehler, die programmatisch behandelt werden können (z. B. eine Karte wurde abgelehnt), enthalten einen Fehlercode, der den gemeldeten Fehler kurz erklärt.


HTTP STATUS CODE SUMMARY
200 - OK Everything worked as expected.
400 - Bad Request The request was unacceptable, often due to missing a required parameter.
401 - Unauthorized No valid API key provided.
402 - Request Failed The parameters were valid but the request failed.
403 - Forbidden The API key doesn't have permissions to perform the request.
404 - Not Found The requested resource doesn't exist.
409 - ConflictThe request conflicts with another request (perhaps due to using the same idempotent key).
429 - Too Many RequestsToo many requests hit the API too quickly. We recommend an exponential backoff of your requests.
500, 502, 503, 504 - Server Errors Something went wrong on myFridge's end. (These are rare.)




User

User Objekte repräsentieren einen Nutzer in der myFridge-App. Mit der API können Sie Ihre User erstellen, löschen und aktualisieren. Sie können sowohl einzelne User als auch eine Liste aller Ihrer User abrufen.

ENDPOINTS
GET
              /api/users
GET
              /api/user/{user_id}
PUT
              /api/user/{user_id}
DELETE
        /api/user/{user_id}




The User Object

THE USER OBJECT
{
    "_id" : "603a27096136a950a84486c5"
    "fridge_id" : "159a2501372365h750a84548d6"
    "shoppinglist_id" : "298z142864028175h750a323225a4"
    "vorname" : "Max",
    "nachname" : "Mustermann",
    "email" : "max.mustermann@mail.de",
    "passwort" : "shj2z384fh2937428hf"
}


Attribute



_id string



Eindeutiger Bezeichner für das Objekt.

fridge_id string

Eindeutiger Bezeichner für das Kühlschrank-Objekt, welcher dem User zugeordnet wird.

shoppinglist_id string

Eindeutiger Bezeichner für das Einkaufslisten-Objekt, welcher dem User zugeordnet wird.

vorname string

Vorname des Users

nachname string

Nachname des Users

passwort string

Passwort des Users. Das Passwort wird bei Erstellung des Objekt gehasht.

email string

E-Mail Adresse des Users

Retrieve a User

METHOD: GET

Um einen User zu bekommen, können Sie nach einem bestimmten User-Object suchen.


Parameter

_id required in url

Die ObjectID des Users.





Update a User

METHOD: PUT

Um bestimmte Daten eines Users ändern zu können, können Sie bestimmte Attribute des jeweiligen User-Objects updaten. Auch wenn nicht alle Daten zeitgleich geändert werden, müssen alle Attribute mitgegeben werden.


Parameter

_id required in url

Der User, bei der die Daten geändert werden sollen.

vorname required

Der neue Vorname des Users.

nachname required

Der neue Nachname des Users.

passwort required

Das neue Passwort.

email required

Die neue E-Mail Adresse des Users.





Delete a User

METHOD: DELETE

Um einen bestimmten User zu löschen, können Sie nach einem bestimmten User-Object suchen und ihn dadurch löschen. Alle Attribute (inklusive der $oid) werden gelöscht.


Parameter

_id required in url

Der User, welcher gelöscht wird.





List all Users

METHOD: GET

Gibt eine Liste der User zurück, die Sie zuvor erstellt haben. Die User werden in sortierter Reihenfolge zurückgegeben, wobei die ältesten User zuerst angezeigt werden.


Parameter

Keine Parameter notwendig.





Allgemein

Die "Session"-Logik besteht darin, dass für einen User, ein Token generiert wird, der für 30 Minuten gültig ist.

Dieser wird bei jeder Abfrage mitgegeben, um Funktionen nutzen zu können, die nur beim Status "eingeloggt" möglich sind.

ENDPOINTS
POST
              /api/registration
POST
             /api/login
GET
               /api/logout




Attribute



Die Attribute sind die User-Attribute. Zu den User-Attributen





Signup

METHOD: POST

Bei der Registrierung wird ein neues User-Objekt erstellt und gleichzeitig ein Fridge-Objekt, welcher dem User zugeordnet wird.



Parameter



vorname required

Der Vorname des neuen Users.

nachname required

Der Nachname des neuen Users.

email required

Die E-Mail Adresse des neuen Users. Es wird nicht explizit geprüft, ob es sich tatsächlich um eine E-Mail Adresse handelt.

passwort required

Das Passwort des neuen Users, welches beim Login mitgegeben wird.





Login

METHOD: POST

Damit der User sich in der App anmelden kann, werden die folgenden Parameter benötigt.


Parameter

email required

Die E-Mail des Users.

passwort required

Das Passwort des Users.





Logout

Der User wird aus seiner vorhanden Session abgemeldet.


Parameter

Keine Parameter notwendig.

Fridge

Fridge Objekte repräsentieren einen Kühlschrank in der myFridge-App. Mit der API können Sie Kühlschränke erstellen, löschen und aktualisieren. Sie können sowohl einzelne Kühlschränke als auch eine Liste aller Ihrer Kühlschränke abrufen.

ENDPOINTS
GET-METHOD
                  /api/fridges
DELETE-METHOD
            /api/fridges
GET-METHOD

POST-METHOD
                /api/fridge
DELETE-METHOD

PUT-METHOD
                  /api/fridge/{fridge_id}
DELETE-METHOD
            /api/fridge/{fridge_id}/{product_name}
